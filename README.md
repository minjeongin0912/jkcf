# 일한문화교류기금 웹사이트 리뉴얼

## 1. 프로젝트 소개

### 프로젝트명

**일한문화교류기금 웹사이트 리뉴얼**
日韓文化交流基金 Website Renewal

### 프로젝트 한 줄 소개

한일 문화교류 사업과 관련 정보를 더욱 쉽고 친근하게 전달할 수 있도록 기존 웹사이트를 현대적으로 개선한 반응형 웹사이트입니다.

### 제작 목적

기존 일한문화교류기금 웹사이트는 다음과 같은 문제점이 있었습니다.

* 사이트의 주요 목적과 사업 내용을 처음 방문한 사용자가 파악하기 어려움
* 작은 글자와 약한 타이포그래피로 인해 정보의 중요도가 명확하게 구분되지 않음
* 텍스트 중심의 단조로운 구성으로 콘텐츠 탐색이 어려움
* 복잡한 메뉴 구조와 좁은 콘텐츠 영역으로 인해 사용자 동선이 불편함
* 페이지별 이미지 비율과 디자인이 일정하지 않아 전체적인 통일감이 부족함
* 활동 사례와 소식 콘텐츠가 다음 행동으로 자연스럽게 연결되지 않음

이러한 문제를 개선하기 위해 파란색과 민트 계열의 색상을 중심으로 공공기관의 신뢰감을 유지하면서도, 넓은 레이아웃과 일러스트를 활용해 친근하고 현대적인 이미지로 리뉴얼했습니다.

또한 정보의 중요도에 따라 제목, 본문, 카드, 버튼의 시각적 위계를 구분하고 메인 페이지에서 주요 사업, 교류 사례, 소식, 영상 콘텐츠로 자연스럽게 이동할 수 있도록 사용자 동선을 개선했습니다.

## 4. 주요 기능

### 4.1 반응형 웹 구현

데스크톱, 태블릿, 모바일 환경에서 콘텐츠를 편리하게 확인할 수 있도록 화면 크기에 따라 레이아웃과 내비게이션 구조가 변경됩니다.

모바일에서는 햄버거 메뉴와 아코디언 형태의 하위 메뉴를 제공해 복잡한 메뉴 구조를 효율적으로 탐색할 수 있도록 구현했습니다.

### 4.2 드롭다운 내비게이션

데스크톱에서는 메뉴에 마우스를 클릭하면 하위 메뉴가 표시됩니다.

모바일에서는 각 대메뉴를 선택하면 관련 하위 메뉴가 펼쳐지며, 다른 메뉴를 선택하면 기존 메뉴가 닫히도록 구현했습니다.

### 4.3 한국어·일본어 전환

페이지의 주요 텍스트를 한국어와 일본어로 전환할 수 있습니다.

선택한 언어는 `localStorage`에 저장되어 다른 페이지로 이동하거나 웹사이트를 다시 방문해도 동일한 언어 설정이 유지됩니다.

### 4.4 메인 비주얼 슬라이더

메인 화면의 주요 공지와 사업 정보를 자동으로 전환되는 슬라이드로 구성했습니다.

* 자동 재생
* 페이지네이션 버튼
* 현재 슬라이드 상태 표시
* 슬라이드별 상세 페이지 이동 버튼

### 4.5 콘텐츠 슬라이더

메인 페이지의 교류 리포트와 유튜브 콘텐츠를 슬라이더 형태로 구현했습니다.

* 이전·다음 버튼
* 페이지네이션
* 화면 크기에 따른 슬라이드 배치 변경
* 유튜브 영상 썸네일 선택 및 영상 재생

### 4.6 뉴스 검색 및 카테고리 필터

뉴스 목록 페이지에서 카테고리를 선택해 원하는 유형의 게시물만 확인할 수 있습니다.

모바일 메뉴의 검색창에 입력한 검색어를 뉴스 페이지로 전달하고, 제목과 내용에 해당 검색어가 포함된 게시물을 표시하도록 구현했습니다.

### 4.7 스크롤 인터랙션

`IntersectionObserver`를 사용해 사용자가 특정 콘텐츠 영역에 진입했을 때 카드와 텍스트가 자연스럽게 나타나도록 구현했습니다.

불필요하게 지속적으로 스크롤 위치를 계산하지 않고, 요소가 화면에 진입하는 시점에만 애니메이션을 실행하도록 구성했습니다.

### 4.8 모바일 공유 기능

모바일 환경에서 현재 페이지를 공유할 수 있도록 Web Share API를 적용했습니다.

Web Share API를 지원하지 않는 환경에서는 현재 페이지 주소가 클립보드에 복사되도록 처리했습니다.

### 4.9 상단 이동 버튼

페이지를 일정 거리 이상 스크롤하면 상단 이동 버튼이 나타나며, 선택 시 페이지 최상단으로 부드럽게 이동합니다.

### 4.10 웹 접근성을 고려한 마크업

* 시맨틱 태그 사용
* 버튼과 메뉴에 `aria-label` 적용
* 메뉴 열림 상태에 `aria-expanded` 적용
* 슬라이드와 메뉴의 활성화 상태 제공
* 키보드 포커스를 통한 드롭다운 메뉴 제어
* `Escape` 키를 이용한 메뉴 닫기

## 6. 문제 해결 과정

### 6.1 복잡한 내비게이션 구조 개선

#### 문제

기존 사이트는 대메뉴와 하위 메뉴의 수가 많아 전체 구조를 한눈에 파악하기 어려웠습니다. 특히 동일한 메뉴 구조를 모바일 화면에 그대로 배치하면 메뉴가 지나치게 길어지고 원하는 페이지를 찾기 어려워지는 문제가 있었습니다.

#### 해결

데스크톱과 모바일의 메뉴 작동 방식을 분리했습니다.

* 데스크톱에서는 마우스 호버와 키보드 포커스로 드롭다운 메뉴 표시
* 모바일에서는 햄버거 메뉴 내부에 아코디언 방식 적용
* 하나의 대메뉴를 펼치면 기존에 열려 있던 메뉴가 닫히도록 처리
* 메뉴가 열렸을 때 배경 스크롤을 제한
* 화면 크기가 변경되면 기존 메뉴 상태를 초기화

이를 통해 동일한 정보 구조를 유지하면서도 각 화면 환경에 적합한 탐색 방식을 제공했습니다.

### 6.2 페이지별로 반복되는 기능 관리

#### 문제

헤더, 언어 전환, 모바일 메뉴와 같은 기능이 여러 서브 페이지에서 반복되기 때문에 각 HTML 파일에 개별적으로 JavaScript를 작성하면 수정과 관리가 어려워지는 문제가 있었습니다.

#### 해결

메인 페이지의 인터랙션은 `index.js`, 서브 페이지의 공통 기능은 `subpage.js`로 분리했습니다.

서브 페이지에서 공통으로 사용하는 다음 기능을 하나의 파일에서 관리했습니다.

* 헤더 스크롤 상태
* 드롭다운 메뉴
* 모바일 메뉴
* 한국어·일본어 전환
* 모바일 공유
* 키보드 접근성

공통 기능을 분리함으로써 중복 코드를 줄이고 유지보수성을 높였습니다.

### 6.3 언어 전환 후 페이지를 이동하면 설정이 초기화되는 문제

#### 문제

단순히 현재 페이지의 텍스트만 변경하면 다른 페이지로 이동하거나 새로고침했을 때 언어가 다시 일본어로 초기화되는 문제가 있었습니다.

#### 해결

각 번역 대상 요소에 `data-ja`와 `data-ko` 속성을 사용하고, 사용자가 선택한 언어를 `localStorage`에 저장했습니다.

페이지가 로드될 때 저장된 언어 정보를 확인해 자동으로 적용하도록 구현하여 전체 페이지에서 일관된 언어 환경을 유지했습니다.

### 6.4 다양한 화면 크기에서 슬라이더가 어긋나는 문제

#### 문제

교류 리포트와 영상 콘텐츠는 데스크톱과 모바일에서 한 화면에 보여야 하는 카드 수와 카드 너비가 달라, 고정된 이동값을 사용하면 일부 카드가 잘리거나 여백이 어긋나는 문제가 있었습니다.

#### 해결

현재 뷰포트와 카드 크기를 기준으로 슬라이드 이동 거리를 계산하고, 화면 크기가 변경될 때 슬라이더의 위치를 다시 계산하도록 구현했습니다.

이전·다음 버튼과 페이지네이션의 현재 상태도 함께 갱신하여 화면 크기와 관계없이 동일한 조작 경험을 제공했습니다.

### 6.5 많은 콘텐츠로 인한 정보 복잡도 개선

#### 문제

재단 소개, 사업 내용, 교류 사례, 뉴스 등 서로 성격이 다른 정보가 텍스트 중심으로 나열되어 사용자가 중요한 내용을 빠르게 파악하기 어려웠습니다.

#### 해결

콘텐츠의 성격에 따라 카드, 목록, 슬라이더, 상세 페이지 등 서로 다른 레이아웃을 적용했습니다.

또한 다음과 같은 시각적 기준을 통일했습니다.

* 제목과 본문의 크기 차이를 통한 정보 위계 설정
* 넓은 콘텐츠 영역과 충분한 섹션 간격 적용
* 동일한 카드 이미지 비율 유지
* 반복되는 버튼의 수를 줄이고 주요 행동만 강조
* 파란색과 민트색을 중심으로 전체 색상 통일
* 곡선과 일러스트를 활용한 친근한 분위기 구성

이를 통해 공공기관 사이트의 신뢰감을 유지하면서도 콘텐츠를 쉽고 편안하게 탐색할 수 있도록 개선했습니다.

## 7. 회고

이번 프로젝트를 통해 단순히 기존 페이지의 외형을 새롭게 바꾸는 것만으로는 좋은 리뉴얼이 될 수 없다는 점을 배웠습니다.

초기에는 각 섹션을 보기 좋게 만드는 데 집중했지만, 작업을 진행하면서 사용자가 사이트에 처음 들어왔을 때 어떤 정보를 먼저 확인해야 하는지, 다음에는 어디로 이동해야 하는지를 함께 고려해야 한다는 점을 알게 되었습니다.

특히 메뉴의 깊이가 깊고 콘텐츠 종류가 많은 사이트에서는 디자인뿐만 아니라 정보 구조와 사용자 동선을 먼저 정리하는 과정이 중요했습니다. 이를 바탕으로 메인 페이지에서 재단 소개, 주요 사업, 교류 사례, 소식으로 자연스럽게 이동할 수 있도록 콘텐츠 순서를 구성했습니다.

반응형 작업에서는 단순히 데스크톱 화면을 작게 축소하는 것이 아니라, 모바일 환경에 맞게 메뉴 방식과 콘텐츠 배치를 새롭게 설계해야 한다는 점을 배웠습니다. 또한 여러 페이지에 반복되는 기능을 공통 JavaScript로 분리하면서 코드 구조와 유지보수의 중요성도 경험했습니다.

다만 순수 HTML, CSS, JavaScript로 여러 페이지를 제작하면서 헤더와 푸터 등 공통 마크업이 각 HTML 파일에 반복되는 한계가 있었습니다. 추후에는 컴포넌트 기반 프레임워크나 템플릿 방식을 사용해 공통 요소를 효율적으로 관리하고, 실제 CMS 데이터와 연동해 콘텐츠를 동적으로 관리할 수 있도록 발전시키고 싶습니다.
