# CURRENT TASK — Header & Navigation

## 현재 작업 단계

이번에는 `HEADER`와 내비게이션만 구현한다.

아래 영역은 절대 구현하지 않는다.

```html
<!-- VISUAL -->
<!-- SECTION 01 -->
<!-- SECTION 02 -->
<!-- SECTION 03 -->
<!-- SECTION 04 -->
<!-- SECTION 05 -->
<!-- FOOTER 01 -->
<!-- FOOTER 02 -->
```

위 영역은 기존 주석 상태 그대로 유지한다.

## 우선 수정 대상

먼저 아래 파일을 확인한다.

- `index.html`
- `css/index.css`
- `js/index.js`

서브페이지 헤더 적용은 사용자가 별도로 요청하기 전까지 미리 진행하지 않는다.

## 헤더 요구사항

### PC 헤더

- PC 기준 디자인은 사용자가 제공한 헤더 참고 이미지와 유사하게 구현한다.
- 헤더 전체 폭은 100%.
- 콘텐츠 이너 최대 폭은 1500px.
- 헤더는 스크롤 시 상단에 고정한다.
- 왼쪽에 로고를 배치한다.
- 로고 경로:

```text
images/logo_header_pc.png
```

- 가운데 또는 오른쪽 영역에 PC 내비게이션을 배치한다.
- 가장 오른쪽에 일본어/한국어 언어 선택 버튼을 배치한다.
- 언어 선택 버튼에는 지구본 아이콘과 아래 방향 표시를 넣는다.
- SVG 아이콘은 HTML 안에 인라인으로 작성해도 된다.
- 외부 아이콘 라이브러리는 사용하지 않는다.

### PC 드롭다운

- 대메뉴에 마우스를 올리면 하위 메뉴가 표시된다.
- 키보드 포커스가 들어와도 하위 메뉴가 표시돼야 한다.
- 열린 드롭다운은 메뉴 아래에 자연스럽게 배치한다.
- 현재 페이지 대메뉴에는 하단 포인트 라인을 표시할 수 있도록 구조를 만든다.
- 드롭다운 항목 호버 시 `#11B1AF` 계열로 강조한다.
- 사용자가 제공한 디자인에 없는 과도한 효과는 넣지 않는다.

## 임시 링크 규칙

현재 페이지 파일:

- 基金について → `about.html`
- 活動内容 → `activities.html`
- 交流事例 → `stories.html`
- お知らせ → `news.html`
- 会員制度について → `membership.html`

하위 메뉴는 해당 페이지의 앵커로 연결한다.

예:

```html
<a href="./about.html#greeting">ご挨拶</a>
<a href="./about.html#history">沿革</a>
<a href="./about.html#officers">役員</a>
```

앵커 대상 섹션은 이번 작업에서 만들지 않는다.

## 한국어 번역

- 기본 표시 언어는 일본어다.
- 언어 선택 버튼에서 일본어와 한국어를 선택할 수 있게 한다.
- 언어 전환 기능은 `js/index.js`에 작성한다.
- 선택한 언어는 `localStorage`로 유지한다.
- HTML 텍스트에는 일본어와 한국어 값을 함께 관리할 수 있는 `data-*` 구조를 사용한다.
- 이번 단계에서 헤더와 내비게이션 문구만 번역한다.

## 모바일

이번 작업에서는 PC 헤더를 먼저 구현한다.

모바일 헤더와 햄버거 메뉴는 사용자가 다음 단계에서 요청할 때 구현한다.

단, HTML과 CSS가 이후 모바일 구현을 방해하지 않도록 기본 구조는 깔끔하게 작성한다.

## 작업 완료 조건

- `index.html`의 `<!-- HEADER -->` 위치에만 헤더가 구현돼 있다.
- 나머지 섹션은 주석만 유지된다.
- `css/index.css`에는 현재 필요한 루트 변수, 기본 스타일, 헤더, PC 내비게이션 스타일만 있다.
- `js/index.js`에는 현재 필요한 언어 선택 및 헤더 기능만 있다.
- 로고 경로가 정확하다.
- 모든 대메뉴와 하위 메뉴에 실제 링크가 있다.
- 드롭다운은 마우스와 키보드로 사용할 수 있다.
- 일본어가 기본값이다.
- 한국어 선택 상태가 저장된다.
- 관련 없는 파일을 수정하지 않는다.
